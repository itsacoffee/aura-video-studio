feat: Add comprehensive E2E tests for video generation pipeline (PR-CORE-003)

Implements comprehensive end-to-end tests validating the complete video generation
workflow from Brief → Script → Voice → Video with real integrations, SSE progress
updates, job queue functionality, and video output quality validation.

## Changes

### New Files
- Aura.E2E/VideoGenerationPipelineE2ETests.cs (~900 lines)
  - 6 comprehensive E2E test scenarios
  - Sophisticated mock providers with real file generation
  - Full pipeline integration testing

- PR_CORE_003_IMPLEMENTATION_SUMMARY.md
  - Detailed implementation documentation
  - Test coverage analysis
  - Technical implementation details

## Test Coverage

### Test 1: Complete Workflow Brief→Script→Voice→Video
- Validates entire pipeline execution end-to-end
- Tracks progress updates through all stages
- Verifies job ID and correlation ID handling
- Confirms successful completion with valid output

### Test 2: VideoOrchestrator with Real Provider Integration
- Uses RuleBasedLlmProvider for actual script generation
- Tests integration with LLM, TTS, Video, and Image providers
- Validates provider method calls and responses
- Confirms script quality and formatting

### Test 3: SSE Progress Updates Validation
- Verifies GenerationProgress event emission
- Validates monotonic progress (never decreases)
- Tests multiple stage reporting (Brief, Script, TTS, Images, Rendering)
- Confirms correlation ID propagation
- Ensures progress reaches completion (>=95%)

### Test 4: Job Queue and Concurrent Video Generation
- Tests VideoGenerationJobService with 3 concurrent jobs
- Validates job creation, queueing, and execution
- Confirms all jobs complete with valid outputs
- Tracks job status transitions

### Test 5: Final Video Output Quality and Format Validation
- Tests 3 quality levels (50, 75, 90)
- Validates 3 format combinations (mp4/H264, mkv/H264, webm/VP9)
- Confirms output format matches RenderSpec
- Verifies resolution and codec handling

### Test 6: Error Handling and Recovery
- Tests invalid input handling
- Validates expected exception types
- Confirms graceful error recovery

## Technical Implementation

### Mock Providers with Real File Generation
- **MockTtsProviderWithFile**: Generates valid WAV files with RIFF headers
- **MockVideoComposerWithFile**: Creates video files with realistic progress updates
- **MockImageProviderWithFiles**: Generates minimal valid JPEG files
- **MockFfmpegLocator**: Simulates FFmpeg detection and validation

### Key Features
- Comprehensive logging with ITestOutputHelper
- Proper resource management with IDisposable
- Realistic system profile detection with fallback
- Automatic cleanup of temporary files
- Progress tracking throughout pipeline
- Correlation ID tracking for distributed tracing

## Running the Tests

```bash
# Run all E2E tests
dotnet test Aura.E2E/Aura.E2E.csproj --filter "FullyQualifiedName~VideoGenerationPipelineE2ETests"

# Run specific test
dotnet test --filter "FullyQualifiedName~VideoGenerationPipelineE2ETests.CompleteWorkflow_BriefToVideo_ShouldSucceed"

# Run with verbose output
dotnet test Aura.E2E/Aura.E2E.csproj --logger "console;verbosity=detailed"
```
